# Plasmo入門

<ShieldBanners socialOnly />

Plasmoへようこそ。Plasmoは、ブラウザ拡張機能開発者が素晴らしい拡張機能を簡単に作成、テスト、公開できるオールインワン・プラットフォームです。Plasmoを使用すれば、面倒な定型コードに別れを告げ、より迅速でシームレスな開発エクスペリエンスを手に入れることができます。

## 製品紹介

当社の主力製品である[Plasmo Framework](#plasmo-framework)は、究極のブラウザ拡張機能SDKです。ブラウザ拡張機能の複雑さを気にせずに、迅速かつ簡単に拡張機能コードを記述できるように設計されています。

拡張機能を構築したら、[Itero TestBed](#itero-testbed)を使用して、現実環境でテストします。この強力なステージング環境では、拡張機能が実際の環境でどのように動作するかを確認し、起動前に必要な変更を加えることができます。レビュープロセスに阻まれることなく、ベータテスターに直接配布できます。

最後に、公開の準備ができたら、[Plasmo BPPツール](#plasmo-browser-platform-publisher)を使用すると、公開プロセスを簡単に自動化できます。数回クリックするだけで、拡張機能を世界中に公開し、ユーザーを獲得し始めることができます。

### Plasmo Framework

ブラウザ拡張機能の構築が簡単で楽しい世界を想像してみてください。面倒な開発プロセスや、複雑なAPIと設定に苦労する時代は終わりました。

Plasmoを使用すると、宣言的な開発が非常に簡単になります。ファイルを書き込み、コンポーネントをエクスポートするだけで、Plasmoが残りの作業（自動的なバンドルとマウント）を行います。さらに、400名を超える開発者からなる[活発なコミュニティ](https://www.plasmo.com/s/d)が常に支援し、専門知識を共有しています。

Plasmoは、TypeScript、React、Preact、Svelte、Vueをファーストクラスでサポートしているため、複雑で美しく、スケーラブルな拡張機能を簡単に構築できます。ライブリロードとReact HMRが組み込まれているため、拡張機能を手動で更新しなくても、変更を即座に確認できます。

Plasmoには、[ストレージ](/framework/storage)と[メッセージング](/framework/messaging)のための強力なライブラリに加え、[.envファイル](/framework/env)の組み込みサポートも含まれています。さらに、Plasmoの「一度学習すれば、どこでも記述可能」なアプローチにより、一度拡張機能を構築すれば、簡単に[複数のブラウザとマニフェストバージョンをターゲットに設定できます](/framework/workflows/build#with-a-specific-target)。

開始するには：[ここをクリック](/framework#getting-started)

移行するには：[ここをクリック](/quickstarts/migrate-to-plasmo)

### Itero TestBed

ブラウザ拡張機能のテストは、面倒で時間のかかるプロセスになる可能性があります。従来の方法では、多くの場合、ブラウザストアからの承認を待つ必要があったり、ベータテスターが手動でアップデートをインストールする必要がありました。Itero TestBedは、ベータテスターに瞬時にアップデートをプッシュできることで、この問題に対するソリューションを提供します。

当社のSaaSプラットフォームはGitHubと統合されており、自動バージョン管理も含まれています。Itero TestBedを使用すると、テストプロセスを合理化し、迅速な反復に集中できます。

機能：

- ベータテスターへのインスタントアップデート
- 簡単なデプロイのためのGitHub統合
- 自動バージョン管理

開始するには：[ここをクリック](/itero)

### Plasmo Browser Platform Publisher (Plasmo BPP)

Chrome、Firefoxアドオン、Microsoft Edgeアドオンストアにブラウザ拡張機能を簡単に公開できるオープンソースのGitHubアクションです。Plasmo BPPを使用すると、公開プロセスを自動化し、拡張機能の構築により多くの時間を費やすことができます。

開始するには：[ここをクリック](/framework/workflows/submit)
