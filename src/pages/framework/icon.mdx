# 拡張機能アイコン

Plasmo フレームワークは、ビルドされたバンドルに対して自動的に低解像度のアイコンバージョンを生成します。そのため、あなたが作業する必要があるのは高解像度バージョンのみです！

デフォルトでは、Plasmo は拡張機能アイコンとして `assets/icon.png` ファイルを読み込みます。必要に応じて、`icon` の代わりにこれらの名前を使用できます。

| icon512 | icon-512 | icon-512x512 | icon1024 | icon-1024 | icon-1024x1024 |
| :------ | :------- | :----------- | -------- | --------- | -------------- |

これらの代替名は、元のアイコンのサイズを明確にするのに役立ちます。ただし、Plasmo は1つのファイルしか選択しません。

<Callout emoji="👉">
  アイコンファイルが `assets` ディレクトリ内にあり、`.png` 拡張子を持つ PNG ファイルであることを確認してください。
</Callout>

## 開発モードアイコン

[開発モード](/framework/workflows/dev)では、Plasmo はアイコンをグレースケールに変換して、本番環境バンドルとの区別を容易にします。

![グレースケールアイコン](@screenshots/extension-icon-example.png)

### カスタム開発モードアイコン

デフォルトのグレースケール処理を望まない場合は、`assets` ディレクトリに `icon.development.png` ファイルを配置できます。Plasmo は開発モードで `icon.png` ファイルの代わりにこのファイルを使用します。

## タグ固有のアイコン

特定の[ビルドタグ](/framework/workflows/build#with-a-custom-tag)に対して異なるアイコンを使用するには、アイコンファイル名を次のようにします。

- `icon.<tag>.png`
- `icon.<tag>.<NODE_ENV>.png`

## アイコンサイズ

Plasmo は次のアイコンサイズ（px単位）を生成します。

| 16x16 | 48x48 | 64x64 | 32x32 | 128x128 |
| ----- | ----- | ----- | ----- | ------- |

`assets` ディレクトリに次のファイルのいずれかを配置することで、特定のサイズを上書きできます。

| 基本                     | 環境依存                        |
| :----------------------- | :---------------------------------- |
| `icon<size>.png`         | `icon<size>.<NODE_ENV>.png`         |
| `icon-<size>.png`        | `icon-<size>.<NODE_ENV>.png`        |
| `icon-<size>x<size>.png` | `icon-<size>x<size>.<NODE_ENV>.png` |
